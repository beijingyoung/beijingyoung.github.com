HTML <a> 标签链接类型

用法：<a rel="链接类型" href="http://example.com/">该链接描述</a>
注意：请勿加入 title 属性
链接类型 描述
nofollow 表明这个链接不是这个站点支持的链接。
external 链接指向外部域，也就是非该域以外的域。
alternate 链接一个当前页面的替代表达方式 如: RSS,RDF,RDFa等等。
author 链接指向该页面作者网络主页。
help 链接指向有关的帮助页面。
license 链接指向当前文档的许可协议。
contents 文档目录
index 文档索引
glossary 术语表
chapter 或者 section 或者 subsection 章节
appendix 附录
start 在系列页面中第一页。
prev 在系列页面中指向该页面的上一页。
next 在系列页面中指向该页面的下一页。
bookmark 书签

构建适用于移动设备样式守则

  • 调整布局文件为纵向一栏，移动设备纵向移动易用性好于横向移动。
  • 移除 CSS :hover 伪类属性， 在触控应用装置中该属性无效果。
  • 减少多余的 空白空间 & padding，小屏幕意味空间更小。
  • 慎用 box-shadow (:none!important;) & opacity 属性，CSS 3 很棒，但是需要大量图形引擎渲染的属性不适合移动设备，这些属性就是移动设备性能杀手！
  • 使用盒子模型 box-ordinal-group 属性 - 该属性可以在不改变标记语言的前提下，灵活改变标记语言的显示数序。
  • 任何图片 alt 属性请使用空值。

搜索引擎优化基本思路

正确理解搜索引擎优化：

“ 正确的搜索引擎优化是指使用正确的优化策略让搜索引擎正确解读这个站点信息与内容，当搜索引擎用户发生关键字搜索行为时，在搜索索引列表中以最合理的内容匹配用户行为。”

SEO关键词优化策略：内容概要，大纲，总结关键字，判断潜在用户搜索关键字行为。

要点：文章关键字个数不要过多，采用正确的语句表达，页面前半部分词组使用极为重要(标题，标签，分类，摘要)。

关键词是否为被优化过 (避开流行度竞争度很高的关键词) 是否可行 尝试问问页面内容 尝试自己为搜索用户  ->  关键字是否获得搜索引擎的高排名？  ->  实测  ->  重构 重写？ 或加入第二搜索词  ->  优化。

IE 和 标准浏览器之间的一些不同之处

对于使用 iframe 元素的文档，针对 IE6/7 的 JavaScript DOM 节点操作，必须保证 DOM 操作就是该文档，否则会抛出 "Invalid Argument" 错误。

使用 JavaScript innerHTML 对文档插入部分内容后，在IE中查看文档，结果为标签元素大写。如果元素标签的属性字段没有空格，属性字段没有双引号。属性字段拥有多个值时，最后一个属性值没有分号。IE 不支持对 <head> 元素赋值操作。

使用 DOM 操作文档的 class 或 style 属性，标准浏览器可以使用 setAttribute() 或 getAttribute() 两个属性进行操作，针对 IE 必须特殊处理。

IE 忽略 CSS 的 white-space: pre-warp; 属性。

IE 解析空 <p></p> 为 <br />，标准浏览器不显示为空的 <p> 标记。

IE 可以直接给对象添加事件，标准浏览器必须使用对象的 setAttribute() 添加事件。
